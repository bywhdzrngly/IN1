# IN1 项目学习路线

## 📚 项目概述
这个项目的核心是 **WebSocket 实时通信 + Flask 后端** 的聊天应用。主要用来学习：
- Flask 后端框架
- WebSocket 双向通信（前后端实时交互）
- SQLAlchemy 数据库 ORM

---

## 第1阶段：理解数据模型和数据库结构
**文件**：`website/__init__.py`

这是整个项目的"基础"，定义了数据结构：

### 四大核心数据模型

| 模型 | 字段 | 说明 |
|------|------|------|
| `User` | id, name, email, password_hash, workspace_list, image | 用户信息（workspace_list 存储加入的 Workspace ID列表） |
| `Workspace` | id, name, admin_username, joining_code | 团队/工作区（joining_code 是4位字母+2位数字的加入码） |
| `Channel` | id, name, admin_username, wid | 频道（属于某个 Workspace，wid 是 Workspace ID） |
| `Chats` | id, message, username, wid, channel_id, image | 聊天消息（image=0 是文本，image=1 是图片） |

**数据关系**：
```
User (多个 Workspace ID)
 ├─ Workspace 1
 │  ├─ Channel 1 → Chats [消息列表]
 │  ├─ Channel 2 → Chats [消息列表]
 │  └─ Channel 3 → Chats [消息列表]
 └─ Workspace 2
    ├─ Channel A → Chats [消息列表]
    └─ Channel B → Chats [消息列表]
```

**关键点**：
- 一个 User 可以加入多个 Workspace（通过 `workspace_list` 字符串存储，用空格分隔 ID）
- 一个 Workspace 有多个 Channel
- 一个 Channel 有多条 Chats 消息

## 第2阶段：理解后端路由和业务逻辑

### 2.1 认证流程 - `website/auth.py`

| 路由 | 方法 | 功能 |
|------|------|------|
| `/signup` | POST/GET | 用户注册（上传头像→保存到 `/uploads` 文件夹） |
| `/login` | POST | 用户登录 |
| `/logout` | GET | 用户注销 |

**流程**：
1. 用户填写注册表单（邮箱、用户名、密码、头像）
2. 上传头像 → 保存到 `uploads/` 文件夹，生成 URL `/uploads/{filename}`
3. 密码使用 `werkzeug.security.generate_password_hash` 加密
4. 创建 User 记录存入数据库

### 2.2 主要视图和上传 - `website/views.py`

| 路由 | 方法 | 功能 |
|------|------|------|
| `/` | GET | 登陆页面 |
| `/authorization` | GET | 登录注册页面 |
| `/chat` | GET | **主聊天页面**（需要登录） |
| `/imageUploadChat` | POST | 上传聊天图片 |

**关键处理**：
- `/chat` 
  - 获取当前用户加入的所有 Workspace
  - 获取第一个 Workspace 的所有 Channel
  - 获取第一个 Channel 的所有聊天记录
  - 将数据传给 Jinja2 模板渲染（初始化前端）

- `/imageUploadChat`
  - 接收上传的图片文件
  - 生成唯一文件名保存到 `uploads/` 文件夹
  - 创建 Chats 记录（`image=1` 标记为图片）
  - 返回重定向到 `/chat`

## 第3阶段：理解 WebSocket 实时通信（核心！）

**文件**：`main.py` 中的 `@socketio.on()` 事件处理

这是**前后端双向通信**的关键。WebSocket 允许后端主动向前端推送消息，实现实时聊天。

### 3.1 连接建立

```python
@socketio.on('connect')
def on_connect():
    # 前端连接时自动触发
```

### 3.2 基础事件

| 事件 | 前后 | 功能说明 |
|------|------|---------|
| `join` | 前→后 | 加入某个 Workspace 房间（基于 Workspace ID） |
| `message` | 前→后 | 发送测试消息（演示用） |

**房间概念**：
- 每个 Workspace 被看作一个"房间"，房间名称是 `Workspace.name`
- 发送消息时，通过 `broadcast=True, room=room.name` 只在该房间广播
- 这样可以隔离不同 Workspace 的消息

### 3.3 Workspace 相关事件

```python
@socketio.on('createWorkspace')
def handle_createWorkspace(data):
    # 前端发送：{username, name}
    # 后端：创建 Workspace，生成加入码，更新 User.workspace_list
    # 后端广播：emit('createWorkspaceJS', {...}, broadcast=True)
    # 前端接收：socket.on('createWorkspaceJS') 更新 Workspace 列表

@socketio.on('joinWorkspace')
def addWorkspace(data):
    # 前端发送：{username, name, code}
    # 后端：验证加入码，更新 User.workspace_list
    # 后端 emit：emit('workspaceJoined', {...}, room=room.name)
    # 前端接收：socket.on('workspaceJoined') 添加到 Workspace 列表
```

**关键理解**：
- 创建时生成 4 位字母+2 位数字的加入码（`random_string(4,2)`）
- 加入时必须提供正确的加入码和 Workspace 名称
- 加入成功后会触发 `join_room(room.name)` 加入 WebSocket 房间

### 3.4 Channel 相关事件

```python
@socketio.on('createChannel')
def handle_createChannel(data):
    # 前端发送：{username, name, wid}
    # 后端：创建 Channel 记录
    # 后端广播：emit('createChannelJS', {...}, room=room.name, broadcast=True)

@socketio.on('getChannels')
def sendChannels(data):
    # 前端发送：{wid}
    # 后端：查询该 Workspace 的所有 Channel
    # 后端 emit：emit('getChannelsJS', {...})
```

### 3.5 聊天消息事件（最重要！）

#### 发送文本消息
```python
@socketio.on('chatmsg')
def chat_msg(data):
    # 前端发送：{msg, username, wid, channel_id}
    # 后端流程：
    # 1. 创建 Chats 记录，image=0（标记为文本消息）
    # 2. 存储到数据库
    # 3. join_room(room.name)
    # 4. emit('receiveMessage', data, broadcast=True, room=room.name)
    
    # 前端接收：
    # socket.on('receiveMessage')
    # 检查当前 channel_id 和 wid 是否匹配
    # 如果匹配，创建聊泡气 DOM 并添加到 #chats 容器
```

#### 获取历史消息
```python
@socketio.on('getMessages')

    # 前端发送：{wid, channel_id}
    # 后端流程：
    # 1. 查询该 Channel 的所有 Chats 记录
    # 2. 组织成 JSON 格式
    # 3. emit('receiveMessageJS', {...}, broadcast=True, room=room.name)
    
    # 前端接收：
    # socket.on('receiveMessageJS')
    # 清空 #chats 容器
    # 遍历返回的消息列表，逐个创建 DOM
    # 区分文本消息（image=0）和图片消息（image=1）
```

#### 通过 WebSocket 发送图片消息

```javascript
    // 1. 建立 WebSocket 连接
    var socket = io.connect('http://127.0.0.1:5000/');
    
    // 2. 连接成功
    socket.on('connect', function() {
        socket.emit('message', {data: 'I\'m connected!'});
        // 获取当前 Workspace 的名称和加入码
        socket.emit('getWorkspaceName', {wid: saved});
    });
});
```

### 前端监听的关键事件

#### 1. 接收新消息
```javascript

        data['wid'].toString() == saved) {

        var div = document.createElement("div");
        div.classList.add('chatbubble');

        } else {
            div.classList.add('chatbubble-left');   // 左对齐 + 显示用户名
```

## 第6阶段：完整流程示例

### 场景：两个用户在同一 Workspace 的 Channel 中聊天

```
【用户1 打开应用】
1. 前端：页面加载 → DOMContentLoaded
2. 前端：socket.connect()
3. 后端：@socketio.on('connect') 触发
4. 前端：已从 /chat 路由获得初始 Workspace 和 Channel 列表
   
【用户1 发送消息 "Hi"】
1. 前端：输入 "Hi" → 点击发送按钮
2. 前端：socket.emit('chatmsg', {
     msg: 'Hi',
     username: 'user1',
     wid: 1,
     channel_id: 5
   })
3. 后端：@socketio.on('chatmsg') 处理
   - 创建 Chats 记录
   - db.session.add(c)
   - db.session.commit()
   - emit('receiveMessage', {
       msg: 'Hi',
       username: 'user1',
       wid: 1,
       channel_id: 5,
       image: 0
     }, broadcast=True, room='Workspace_Name')
     
4. 前端（用户1 和 用户2 同时收到）：socket.on('receiveMessage')
   - 检查：channel_id == 5 && wid == 1 ✓
   - 创建聊泡气 DOM
   - 添加到 #chats 容器
   - scrollDownChatWindow() 自动滚到底部
   
【用户2 上传一张图片】
1. 前端：点击图片按钮 → 选择文件
2. 前端：表单提交到 POST /imageUploadChat
3. 后端：uploadImage()
   - 保存图片到 /uploads 文件夹
   - 创建 Chats 记录（image=1）
   - db.session.commit()
   - 重定向到 /chat
   
4. 前端：socket.emit('sendimage') 通过 WebSocket 广播
5. 后端：@socketio.on('sendimage')
   - emit('receiveMessage', {
       message: '/uploads/xxx.jpg',
       image: 1,
       ...
     }, broadcast=True, room='Workspace_Name')
     
6. 前端（用户1 和 用户2）：socket.on('receiveMessage')
   - 检查 image == 1
   - 创建 <img> 标签
   - 添加到聊泡气
```

## 🎯 核心概念速记

| 概念 | 说明 |
|------|------|
| **Room** | WebSocket 房间 = Workspace 名称，用于隔离不同 Workspace 的消息 |
| **broadcast** | `broadcast=True` 表示消息发送给房间内的所有连接用户 |
| **emit** | 后端向前端发送数据的方法 |
| **socket.on vs emit** | on = 监听；emit = 发送 |
| **image 字段** | 0 = 文本消息；1 = 图片消息 |
| **workspace_list** | User 模型中的字符串字段，存储用空格分隔的 Workspace ID |
| **joining_code** | 4 位字母 + 2 位数字，用于邀请他人加入 Workspace |

## 改进建议（前端重做时）

1. **框架升级**：不用 Jinja2 模板，改用 Vue.js / React 组件化
2. **消息流管理**：使用事件总线或状态管理库（Vuex / Redux）
3. **UI/UX 改进**：重新设计界面，支持响应式设计
4. **性能优化**：
   - 虚拟滚动（显示很多消息时）
   - 消息分页加载
   - 连接重连机制
5. **功能扩展**：
   - 消息编辑/删除
   - 用户在线状态
   - 打字中提示
   - 消息已读标记
   - 文件分享（暂不仅限图片）
